## 三、 API 拦截器与错误码对接 (API Layer)

### 3.1 Axios 实例范式 (`api/client.ts`)

> **重要约束**：拦截器属于 API 层，遵循 §2.2 "严禁包含 UI 逻辑" 原则。  
> 错误提示（`ElMessage` 等）统一由 **Composable 层的 `catch` 块** 处理，拦截器仅负责解包、鉴权跳转和 `reject`。

```typescript
import axios from 'axios'

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080/api/v1',
  timeout: 12000,
})

// ── 请求拦截器：注入 Token ──
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('tp-token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// ── 响应拦截器：解包（严禁 UI 逻辑） ──
apiClient.interceptors.response.use(
  (response) => {
    const body = response.data
    // 自动解包 { code, message, data } 结构
    if (body && typeof body === 'object' && 'code' in body && 'data' in body) {
      // 分页响应统一化
      if ('total' in body && 'page' in body) {
        response.data = {
          items: body.data,
          total: body.total,
          page: body.page,
          pageSize: body.page_size,
        }
      } else {
        response.data = body.data
      }
    }
    return response
  },
  (error) => {
    // 401 → 清除凭证并跳转登录（唯一允许的副作用）
    if (error.response?.status === 401) {
      localStorage.removeItem('tp-token')
      window.location.href = '/login'
    }
    // 其他错误一律 reject，由 Composable 层决定如何提示用户
    return Promise.reject(error)
  },
)

export { apiClient }
```

## 五、 组件开发规范 (Component Rules)

### 5.1 SFC 块顺序 (强制)
```
<script setup lang="ts">  ←  第一
<template>                ←  第二
<style scoped>            ←  第三
```

### 5.2 Props 与 Emit 类型化 (强制)
```vue
<script setup lang="ts">
/**
 * StatusBadge - 通用状态标签组件
 * 根据 status 值显示不同颜色的标签
 */
interface Props {
  status: 'pending' | 'approved' | 'rejected'
  label?: string
}

const props = defineProps<Props>()
const emit = defineEmits<{
  (e: 'click', status: string): void
}>()
</script>
```

### 5.3 代码行数控制

| 区域 | 上限 | 超出时 |
|------|------|--------|
| `<script>` | 200 行 | 提取 `useXxx` composable |
| `<template>` | 100 行 | 拆分子组件 |
| 组件总计 | 300 行 | 强制拆分 |

### 5.4 异步三态闭环 (强制)
```vue
<template>
  <!-- Loading 态 -->
  <el-skeleton v-if="loading" :rows="5" animated />

  <!-- Empty 态 -->
  <EmptyState v-else-if="!items.length" description="暂无评审计划" />

  <!-- Data 态 -->
  <el-table v-else :data="items">
    <!-- 表格列 -->
  </el-table>
</template>
```

### 5.5 `<style scoped>` 与子组件拆分 (陷阱警示)

Vue 的 `<style scoped>` **不会穿透到子组件内部 DOM**。当你按照 §5.3 将 template 拆分为子组件时，**必须同步迁移相关样式**到子组件中。

```
⚠️ 典型事故：
  父组件 TagManagement.vue 的 scoped 样式中定义了 .tm-heat-grid 的 grid 布局，
  提取 TagHeatGrid.vue 子组件后，grid 布局完全失效，页面变为无样式的文字堆叠。
```

**规则**：
- 拆分子组件时，将该组件使用的 CSS 类一并移至子组件 `<style scoped>`
- 父组件只保留自身 template 直接使用的样式
- CSS 变量 (Custom Properties) 可正常继承，无需重复声明
- 需要穿透第三方组件样式时使用 `:deep()` 选择器

### 5.6 Composable 层错误提示 (强制)

API 层严禁 UI 逻辑（§3.1），错误提示统一在 Composable 的 `catch` 块处理：

```typescript
async function loadData() {
  loading.value = true
  try {
    const result = await listItems(projectId)
    items.value = result.items
  } catch (err: unknown) {
    // 错误提示在 Composable 层处理，不在 API 层
    const msg = (err as any)?.response?.data?.message || '加载失败'
    ElMessage.error(msg)
  } finally {
    loading.value = false
  }
}
```

## 六、 Composable 范式 (Business Logic)

### 6.1 useTable 范式
```typescript
import { ref } from 'vue'

interface PageResult<T> {
  items: T[]
  total: number
}

/**
 * 表格通用逻辑：分页、加载态、刷新
 * @param fetchFn - API 函数，接受 page/pageSize 返回分页结果
 */
export function useTable<T>(
  fetchFn: (params: { page: number; pageSize: number }) => Promise<PageResult<T>>
) {
  const items = ref<T[]>([]) as Ref<T[]>
  const total = ref(0)
  const page = ref(1)
  const pageSize = ref(20)
  const loading = ref(false)

  async function loadData() {
    loading.value = true
    try {
      const result = await fetchFn({ page: page.value, pageSize: pageSize.value })
      items.value = result.items
      total.value = result.total
    } finally {
      loading.value = false
    }
  }

  function handlePageChange(newPage: number) {
    page.value = newPage
    loadData()
  }

  return { items, total, page, pageSize, loading, loadData, handlePageChange }
}
```

## 七、 Store 设计规范 (Pinia)

### 7.1 按领域拆分
```typescript
// stores/auth.ts
export const useAuthStore = defineStore('auth', () => {
  const token = ref<string | null>(localStorage.getItem('tp-token'))
  const user = ref<User | null>(null)
  const isAuthenticated = computed(() => !!token.value)

  async function login(email: string, password: string) {
    const res = await loginByEmail(email, password)
    token.value = res.data.token
    user.value = res.data.user
    // 持久化到 localStorage
    localStorage.setItem('tp-token', res.data.token)
  }

  function logout() {
    token.value = null
    user.value = null
    localStorage.removeItem('tp-token')
  }

  return { token, user, isAuthenticated, login, logout }
})
```

### 8.3 暗色主题策略

页面级暗色主题通过**根元素 CSS 变量覆盖**实现，而非散落的硬编码颜色值：

```css
/* 暗色页面的根元素覆盖全局 token */
.page-dark-root {
  --tp-primary: #e1e1f2;
  --tp-primary-light: #d2bbff;
  --bg-main: #11131e;
  --bg-card: #191b26;
  /* ... 其余覆盖 */
}
```

**规则**：
- 组件内部严禁出现 `#11131e` 等硬编码暗色值，统一引用 CSS 变量
- 暗色主题的变量集应集中定义在页面根组件或 `styles/dark.css`
- 未来可通过 `prefers-color-scheme` 或用户偏好自动切换

 * 标签管理业务逻辑 Composable

 * 封装标签的 CRUD、批量创建、搜索、分页等全部业务逻辑，
 * 供 TagManagement.vue 页面编排层消费。
 *
 * @module useTagManagement
 */
export function useTagManagement() {
  // ── 响应式状态 ──
  const tags = ref<Tag[]>([])       // 当前页标签列表
  const loading = ref(false)        // 加载状态
  const total = ref(0)              // 总条数（分页用）

  /**
   * 加载标签列表（支持关键词搜索 + 分页）
   * 内部使用 AbortController 防止搜索竞态
   *
   * @param keyword - 搜索关键词（可选）
   * @param page    - 页码，默认 1
   */
  async function loadTags(keyword?: string, page = 1) {

    controller = new AbortController()

    loading.value = true
    try {
      const resp = await listTags(projectId, { keyword, page })
      tags.value = resp.items    // 后端返回的标签数组
      total.value = resp.total   // 总条数
    } catch (err) {
      // 忽略主动取消的请求错误
      if (!isAbortError(err)) {
        ElMessage.error('加载标签失败')
      }
    } finally {
      loading.value = false
    }
  }

  // ── 批量创建 ──
  // 按逗号或空格分割输入，为每个标签分配互斥色后批量调用创建 API
  async function batchCreate() { /* ... */ }

  return { tags, loading, total, loadTags, batchCreate }
}

## 十四、 无障碍标准 (Accessibility / a11y)

> 工业级产品必须满足 **WCAG 2.1 Level AA** 基线。

### 14.1 强制规则

| 规则 | 示例 |
|------|------|
| 所有可交互元素必须可键盘聚焦 | `<button>` 而非 `<div @click>` |
| 图片必须有 `alt` 属性 | `<img :alt="tag.name" />` |
| 表单控件必须关联 `<label>` | Element Plus `<el-form-item label="...">` 已满足 |
| 颜色不作为唯一信息通道 | 状态标签需同时显示文字（如"使用中"）|
| 对比度 ≥ 4.5:1（正文）/ 3:1（大字） | 使用 Chrome DevTools 检查 |

### 14.2 ARIA 增强范式
```vue
<!-- ✅ 自定义组件加 ARIA -->
<div role="tablist" aria-label="标签切换">
  <button
    v-for="tab in tabs" :key="tab.id"
    role="tab"
    :aria-selected="activeTab === tab.id"
    @click="activeTab = tab.id"
  >{{ tab.label }}</button>
</div>

<!-- ✅ 加载状态通知屏幕阅读器 -->
<div aria-live="polite" aria-busy="loading">
  <el-skeleton v-if="loading" />
  <div v-else>{{ content }}</div>
</div>
```

## 十六、 大列表性能优化 (Virtual Scrolling)

当列表项超过 **100 条**时，必须使用虚拟滚动避免 DOM 节点过多导致卡顿：

```vue
<script setup lang="ts">
import { useVirtualList } from '@vueuse/core'

const { list, containerProps, wrapperProps } = useVirtualList(items, {
  itemHeight: 72,
})
</script>

<template>
  <div v-bind="containerProps" style="height: 600px; overflow-y: auto">
    <div v-bind="wrapperProps">
      <div v-for="{ data } in list" :key="data.id" style="height: 72px">
        {{ data.name }}
      </div>
    </div>
  </div>
</template>
```

**适用场景**：标签列表、用例列表、评审项列表等可能超过 100 条的数据展示。
